## 3. 交付目标：优先生成可公开分享的网站

### 3.1 第一优先级：可直接通过链接访问

优先生成一个可部署的静态网站项目，并确保可以部署到以下任一平台：

- GitHub Pages
- Vercel
- Netlify
- Cloudflare Pages

项目至少包含：

- `index.html`
- `README.md`
- 必要的部署配置文件
- 完整静态资源
- 清晰的部署步骤

如果当前环境具备 Git、GitHub、Vercel、Netlify 或 Cloudflare 的有效登录状态和权限，可以执行部署，并在成功后输出真实公开网址。

**禁止虚构部署成功。禁止编造网站链接。**

如果缺少账号、Token、网络权限或部署权限：

1. 仍然完整生成可部署项目。
2. 生成最简明的部署说明。
3. 明确指出缺少哪一步权限。
4. 同时生成本地可用的单文件版本。

### 3.2 第二优先级：单文件本地 HTML

无论是否成功部署，都必须额外交付：

- `dist/shuchenglin-ai-board.html`

该文件必须：

- 双击即可打开
- 不依赖后端
- 不依赖 npm
- 不依赖必须联网加载的字体、图片或 CDN
- 所有 CSS 和 JavaScript 内嵌
- 使用内联 SVG 或 CSS 绘制图形
- 适配电脑、平板和手机
- 可直接发给别人打开观看
- 所有核心交互都能演示

### 3.3 推荐项目结构

```text
/
├─ CLAUDE.md
├─ index.html
├─ README.md
├─ vercel.json                 # 如采用 Vercel
├─ netlify.toml                # 如采用 Netlify，可选
├─ .nojekyll                   # 如采用 GitHub Pages，可选
└─ dist/
   └─ shuchenglin-ai-board.html
```

## 4. 技术原则

优先使用原生 HTML、CSS、JavaScript。

对于本项目 Demo：

- 不使用 React、Vue 或复杂框架
- 不要求 npm 安装
- 不依赖构建工具才能运行
- 不依赖外部 CDN
- 不依赖联网字体
- 图标使用内联 SVG
- 森林、叶片、光影、年轮使用 CSS 和内联 SVG 实现
- 示例头像使用文字头像、抽象渐变或内联 SVG
- 不使用未经授权的真实人物照片
- 所有示例人物和项目必须明确为虚构数据
- 使用 `localStorage` 保存演示状态
- 所有 JavaScript 必须无控制台报错
- 代码结构要方便未来接入真实数据库与 AI API

如果为了部署需要建立多文件项目，也必须同步生成完整单文件版。

## 7. 核心模块二：AI 评审官

模块名称：

**AI 评审官**

英文名称：

**AI Judge**

核心文案：

**让作品先被理解，再被评价。**

AI 评审官承担：

- 读取作品资料
- 分析成果证据
- 根据公开标准结构化评分
- 检查材料完整度
- 识别疑似重复作品
- 标注可能存在的虚假或夸大描述
- 提供可解释评分理由
- 对不同赛道使用不同评分模型
- 帮助人工评委快速理解作品
- 发现排名较低但潜力较高的项目
- 推荐适合的导师、成员和合作资源
- 为创作者生成改进路线

AI 不得作为唯一裁判。

正式榜单权重：

- AI 结构化评审：30%
- 专业评委评审：30%
- 认证社群成员评议：25%
- 可验证成果证据：15%

页面必须明确：

**AI 负责提高评审效率，但不拥有对一个人的最终定义权。**

**任何重要判断都应该可以解释、可以复核、可以申诉。**

AI 评分详情展示：

- 评分标准
- 每项得分
- 评分理由
- 引用的作品证据
- AI 置信度
- 可能存在的偏差
- 人工复核状态
- 模型版本
- 评分时间
- 申诉入口

AI 的人格应当认真、理性、谦逊，而不是永远正确的绝对权威。

## 8. 核心模块三：AI 公平性与透明中心

设置：

**AI 评审透明中心**

展示：

- 当前评审规则
- 评分维度
- 榜单权重
- AI 模型版本
- 人工复核比例
- 申诉流程
- 异常投票数量
- 无效票说明
- 规则更新时间

公平原则：

1. 不以粉丝数量作为能力指标。
2. 不以社交关系决定曝光。
3. 不以学校知名度自动加分。
4. 不以城市、性别、专业背景决定评分。
5. 不因包装精美而忽视真实效果。
6. 不因项目规模较小而否定社会价值。
7. AI 低置信度结果必须进入人工复核。
8. 候选人可以申请重新评分。
9. 正式结果保留完整审计记录。
10. 算法推荐必须显示推荐理由。

反偏见文案：

**我们希望 AI 帮助普通人的作品被看见，而不是让已有优势的人获得更多优势。**

## 9. 核心模块四：潜力森林推荐系统

推荐流名称：

**潜力森林**

不按照单纯热度排序，而是发现：

- 作品质量高但曝光较少的人
- 持续进步明显的人
- 首次提交的新成员
- 来自非热门城市和高校的优秀作品
- 解决小众但重要问题的作品
- 对社群帮助很多但不善表达的人
- 有真实用户价值但传播能力较弱的项目
- 具有跨领域合作潜力的项目

每个推荐卡片展示推荐原因，例如：

- 因为持续三周更新而被推荐
- 因为解决了一个被忽视的问题而被推荐
- 因为获得真实用户反馈而被推荐
- 因为帮助了 12 位社群成员而被推荐
- 因为项目进步幅度进入本周前 10% 而被推荐
- 因为与你的能力形成互补而被推荐

算法价值观：

**不是让最热门的人越来越热门，而是让真正有价值的创造有机会抵达需要它的人。**

## 17. 投票与评议机制

核心原则：

**投票打榜，不搞拉票。**

规则：

1. 只有完成身份验证的社群成员可以参与正式评议。
2. 每位成员每期拥有固定数量的评议票。
3. 评议前必须查看作品与成果证据。
4. 候选人顺序随机显示。
5. 投票阶段隐藏实时票数。
6. 每次评议需要填写或选择理由。
7. 不生成候选人专属拉票链接。
8. 分享只能进入完整榜单或项目详情。
9. 不允许给自己投票。
10. 粉丝数、群人数和转发量不计入评分。
11. 异常集中投票进入人工复核。
12. 批量重复理由视为无效评议。
13. 结果公布后公开有效票、无效票和复核情况。
14. 提供申诉入口。

投票成功文案：

**你的判断，正在帮助一项真实创造被看见。**

使用新叶生长动效，不使用烟花、金币或夸张庆祝。
